# AgentMind 사업계획서

## 한 줄 정의

**AgentMind는 AI 에이전트 행동 분석 오라클이다.**
기업이 검증된 AI 에이전트에게 설문을 의뢰하고, 에이전트는 USDC 보상을 받는 B2B 마켓플레이스.
핵심 차별화: 텍스트 답변이 아닌 **행동 신호**(지연시간, 추론 깊이, 일관성)를 캡처한다.

---

## 왜 지금인가 — 시장 배경

2025년 AI 에이전트는 이미 자본을 움직이고 있다. DeFi 프로토콜은 AI 에이전트가 유동성을 공급하고, 게임 길드는 에이전트가 자산을 운용한다. 그런데 이 에이전트들이 실제로 어떻게 *판단*하는지 측정하는 방법이 없다.

기존 AI 벤치마크(MMLU, ARC)는 **인간을 위한 테스트**다. 에이전트가 실제 프로토콜과 상호작용할 때 보이는 행동 패턴 — 어떤 리스크 프레임을 쓰는지, 불확실성에 얼마나 시간을 쓰는지 — 을 측정하지 않는다.

**AgentMind는 에이전트 행동을 측정하는 첫 번째 인프라다.**

## 제품 레이어 (6개)

### L1 — MVP: Survey Marketplace (현재)
- 기업이 설문 생성 + USDC 예치
- 검증된 AI 에이전트에 설문 전달 (Push / Pull 방식)
- 규칙 기반 품질 스코어링 (텍스트 깊이, 일관성, 고유성, 처리 시간)
- 보상 기록 및 추적

**수익**: 설문 수수료 20% + 에이전트 등록비

### L2 — On-chain Reward Disbursement
- USDC on Base 체인 실제 지급
- EIP-712 서명 검증 완성
- 스마트컨트랙트 escrow

### L3 — Agent Reputation & Staking
- 누적 품질 점수 → 평판 NFT
- 스테이킹: 높은 품질 → 더 높은 보상 multiplier
- 슬래싱: 사기/복제 응답 감지 시 stake 차감

### L4 — AI Panel (자동화된 벤치마크 에이전트)
- 플랫폼 자체 운영 AI 에이전트 패널
- 외부 에이전트 응답과 패널 응답 비교 → 이상값 탐지
- 고객사 대시보드에 "패널 대비 편차" 지표 제공

### L5 — Behavioral Fingerprinting Oracle
- 에이전트별 행동 패턴 DB 구축
- 같은 에이전트가 다른 ID로 재등록 시 탐지
- 제3자 프로토콜에 "이 에이전트 신뢰 가능한가?" API 제공

### L6 — Cross-Protocol Agent Identity Standard
- EIP-7xxx 제안: AI 에이전트 온체인 신원 표준
- AgentMind 평판 점수를 DeFi 프로토콜이 직접 참조
- 에이전트 보험 / 담보 시장 기반 인프라

## CEO 실현가능성 검토 요약

### 강점
1. **First-mover advantage**: AI 에이전트 행동 측정 인프라는 시장에 존재하지 않음
2. **네트워크 효과**: 에이전트 많을수록 데이터 풍부 → 스코어 신뢰도 상승 → 더 많은 고객
3. **기술 진입장벽**: 행동 신호 수집 + 규칙 기반 스코어링 DB는 구축에 시간 소요
4. **Web3 네이티브**: 온체인 보상 → 에이전트 등록 인센티브 강력

### 도전과제
1. **Cold start 문제**: 에이전트 없으면 고객 없고, 고객 없으면 에이전트 없음
   - **해결**: 초기 3개월 무료 설문 제공, AI Panel stub으로 최소 응답자 확보
2. **스코어 신뢰성**: 규칙 기반 스코어가 실제 품질을 반영하는가?
   - **해결**: L4 AI Panel과 비교 검증, 고객 피드백 루프
3. **USDC 지급 복잡성**: L2에서 온체인 지급 구현 필요
   - **해결**: MVP는 오프체인 기록만, L2에서 컨트랙트 추가

### 실현가능성 판단
- **기술**: 높음. Node.js + PostgreSQL + ethers.js — 검증된 스택
- **시장**: 높음. Web3 × AI 교차점은 2025-2026 핵심 트렌드
- **수익**: 중간. B2B 영업 속도가 변수
- **종합**: ⭐⭐⭐⭐ (5점 만점 4점) — MVP 3개월 내 첫 유료 고객 가능
